\section{Background and Motivation}

The Pension Fund Efficiency Toolkit is a software prototype developed in support of the
research study \textit{Modeling Efficiency of Self-Administered Pension Funds in Zimbabwe
Using a Frontier-Based Optimization Analysis and Random Forest Model}. The study applies
Data Envelopment Analysis (DEA) to evaluate the technical efficiency of Zimbabwean
self-administered standalone pension funds over the period 2018--2022, and subsequently
employs a Random Forest regression model to identify the environmental determinants of
efficiency differentials.

The prototype operationalises the full DEA-plus-machine-learning pipeline in Python~3.11,
using \texttt{PuLP} for linear programming, \texttt{scikit-learn} for machine learning,
\texttt{joblib} for parallel bootstrap computation, \texttt{Streamlit} for an interactive
dashboard, and \texttt{ReportLab} for automated PDF report generation.

\section{Objectives of the Prototype}

\begin{enumerate}[noitemsep]
  \item Implement input-oriented CCR (CRS) DEA to evaluate overall technical efficiency.
  \item Implement input-oriented BCC (VRS) DEA to isolate pure technical efficiency.
  \item Compute scale efficiency and classify each fund by IRS/CRS/DRS regime.
  \item Apply Simar-Wilson (1998) bootstrap bias correction with 95\% confidence intervals.
  \item Fit a Random Forest second-stage regression on six contextual determinants.
  \item Provide a Streamlit dashboard for interactive analysis and PDF export.
  \item Support reproducibility through Docker containerisation and GitHub Actions CI.
\end{enumerate}

\section{Scope}

The prototype operates on panel data from 16--20 Zimbabwean self-administered standalone
pension funds, each observed over five annual periods (2018--2022). Data are aggregated
to fund-level means before cross-sectional DEA is executed. The system does not currently
implement window DEA, Malmquist indices, or stochastic frontier analysis.

\section{DEA Matrix Construction}

Multi-year panel data are reduced to a cross-section by computing fund-level arithmetic
means across 2018--2022.  The DEA matrices extracted are:

\[

\text{OpEx}_1 & \text{Assets}_1 & \text{Equity}_1\\
\vdots&\vdots&\vdots\\
\text{OpEx}_n & \text{Assets}_n & \text{Equity}_n
\end{pmatrix}\in\mathbb{R}^{n\times3},\quad
Y = \begin{pmatrix}
\text{NetInvInc}_1 & \text{Contrib}_1\\
\vdots&\vdots\\
\text{NetInvInc}_n & \text{Contrib}_n
\end{pmatrix}\in\mathbb{R}^{n\times2}
\]

\setcounter{section}{0}\renewcommand{\thesection}{4.\arabic{section}}

\section{Mathematical Formulation}

\subsection{CCR (CRS) Input-Oriented Model}

For each DMU $k$ ($k=1,\ldots,n$), solve:
\begin{align}
\min_{\theta,\boldsymbol{\lambda}}\;&\theta \label{eq:ccr}\\
\text{s.t.}\;&\sum_j\lambda_j x_{ij}\le\theta x_{ik},\quad i=1,\ldots,m\notag\\
&\sum_j\lambda_j y_{rj}\ge y_{rk},\quad r=1,\ldots,s\notag\\
&\lambda_j\ge0\notag
\end{align}
$\theta\in(0,1]$ is the radial efficiency score.  A score of 1 indicates the fund lies on
the efficiency frontier; $1-\theta$ is the proportional input reduction achievable at the
same output levels.

\subsection{BCC (VRS) Extension}

Adding the convexity constraint $\sum_j\lambda_j=1$ to~\eqref{eq:ccr} gives the BCC model,
which allows variable returns to scale.  The BCC score $\theta^\text{BCC}\ge\theta^\text{CCR}$.

\subsection{Two-Phase Slack Maximisation}

Phase I solves~\eqref{eq:ccr} to obtain $\theta^*$.
Phase II fixes $\theta=\theta^*$ and maximises $\sum_i s_i^- + \sum_r s_r^+$ with equality
constraints and non-negative slacks, eliminating weak efficiency.

\section{Scale Efficiency}

\begin{equation}
\mathrm{SE}_k = \frac{\theta_k^\text{CCR}}{\theta_k^\text{BCC}},\qquad
\mathrm{RTS}_k=
\begin{cases}\text{IRS}&\sum_j\lambda_j^*<1\\\text{CRS}&\sum_j\lambda_j^*\approx1\text{ and }\mathrm{SE}\approx1\\\text{DRS}&\sum_j\lambda_j^*>1\end{cases}
\end{equation}

\begin{center}{\LARGE\bfseries CHAPTER 5}\\[0.5cm]{\Large\bfseries Simar-Wilson Bootstrap Bias Correction}\end{center}
\setcounter{section}{0}\renewcommand{\thesection}{5.\arabic{section}}
\phantomsection\addcontentsline{toc}{section}{\textbf{CHAPTER 5: SIMAR-WILSON BOOTSTRAP}}\vspace{1cm}

\section{Motivation}

DEA scores are upward-biased estimators of true efficiency because the frontier is
estimated from the same sample used for evaluation.  Simar and Wilson (1998) Algorithm~1
corrects this bias and constructs valid confidence intervals.  The implementation in
\texttt{bootstrap.py} parallelises $B=2{,}000$ replications with \texttt{joblib}.

\section{Algorithm}

\begin{enumerate}[noitemsep]
  \item Compute original DEA scores $\hat{\boldsymbol{\theta}}$.
  \item Estimate Silverman bandwidth:
    $h = 0.9\min\!\bigl(\hat\sigma_\theta,\,\tfrac{\text{IQR}_\theta}{1.34}\bigr)n^{-1/5}$.
  \item For $b=1,\ldots,B$ (parallelised):
    \begin{enumerate}[noitemsep,label=(\alph*)]
      \item Draw $\boldsymbol{\theta}^*$ from reflected KDE of $\hat{\boldsymbol{\theta}}$;
            reflect values $>1$ as $2-\theta^*$.
      \item Pseudo-inputs: $X^* = \mathrm{diag}(\boldsymbol{\theta}^*/\hat{\boldsymbol{\theta}})\,X$.
      \item Solve DEA on $(X^*,Y)\to\hat{\boldsymbol{\theta}}^{*b}$.
    \end{enumerate}
  \item Bias: $\hat B_k = \overline{\hat\theta^*_k}-\hat\theta_k$.
  \item Bias-corrected: $\tilde\theta_k=\hat\theta_k-\hat B_k$.
  \item 95\% CI from empirical bootstrap percentiles.
\end{enumerate}

\section{Two-Stage Framework}

Stage~1 (DEA) produces bias-corrected scores $\tilde\theta_k$.  Stage~2 regresses these
scores on six contextual variables using a Random Forest regressor to quantify which
structural and environmental factors drive efficiency differentials.

\section{Model Configuration}

\textbf{RandomForestRegressor}: 500 trees, \texttt{max\_features="sqrt"}, fixed seed.\\
\textbf{Feature importance}: MDI (built-in) + permutation importance (30 repeats).\\
\textbf{Validation}: $k$-fold cross-validation ($k\le5$), reporting mean $\pm$ std R$^2$.\\
\textbf{Visualisation}: Partial Dependence Plots (PDPs) for top-3 features.

\section{Peer Analysis and Benchmarking}

For each inefficient DMU, the toolkit identifies its \emph{peer set} --- the subset of
efficient frontier funds that serve as best-practice benchmarks.  Peers are defined as
funds with $\lambda_j^* > 10^{-4}$ in the Phase~II optimal solution.  The input reduction
target for each inefficient fund $k$ is:
\[
x_{ik}^\text{target} = \theta_k^* \cdot x_{ik} - s_{ik}^{-*},\quad
\text{reduction}_{ik} = x_{ik} - x_{ik}^\text{target}
\]
These targets are exported in \texttt{out/targets.csv} and displayed in the Recommendations
dashboard tab, enabling fund managers to identify concrete operational improvement paths.

\section{Summary of Prototype Contributions}

The Pension Fund Efficiency Toolkit delivers a complete, reproducible, and modular
software prototype that operationalises the DEA-plus-Random-Forest analytical framework
for evaluating Zimbabwean self-administered pension funds.  The principal contributions
of the prototype are:

\begin{enumerate}[noitemsep]
  \item A two-phase input-oriented DEA engine implementing both CCR (CRS) and BCC (VRS)
        models via PuLP/CBC, producing efficiency scores, input slacks, and peer
        identification for up to 20 fund DMUs.
  \item A scale efficiency module classifying each fund's returns-to-scale regime (IRS,
        CRS, DRS) to distinguish managerial from scale-related inefficiency.
  \item A parallelised Simar-Wilson bootstrap bias-correction engine producing statistically
        valid efficiency estimates with 95\% confidence intervals.
  \item A Random Forest second-stage regression explaining efficiency score variation using
        six contextual determinants, with MDI, permutation importance, and PDP outputs.
  \item An interactive Streamlit dashboard enabling non-technical users to upload data,
        execute the pipeline, explore results, and export PDF reports without writing code.
  \item A containerised, CI-tested deployment ensuring reproducibility across environments.
\end{enumerate}

\section{Limitations and Future Work}

The current prototype executes cross-sectional DEA on fund-level means, aggregating the
2018--2022 panel to a single time point.  Future development should incorporate:

\begin{itemize}[noitemsep]
  \item \textbf{Window DEA}: Rolling time-window analysis to track efficiency changes over
        individual years without full panel aggregation.
  \item \textbf{Malmquist Productivity Index}: Decomposing total factor productivity change
        into efficiency change and frontier shift components.
  \item \textbf{Tobit second stage}: As a parametric alternative to the Random Forest for
        robustness comparison.
  \item \textbf{Super-efficiency DEA}: To rank frontier funds and resolve ties among
        efficient DMUs.
  \item \textbf{Real IPEC data integration}: Direct API or file-feed integration with IPEC
        regulatory disclosures to replace synthetic data with official audited figures.
\end{itemize}
